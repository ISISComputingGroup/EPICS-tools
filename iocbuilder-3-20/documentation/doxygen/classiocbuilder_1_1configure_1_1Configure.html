This is the lowest level IOC builder initialisation function.

Typically this function should not be called directly, instead one of its proxies, ConfigureIOC(), ConfigureTemplate() or ParseEtcArgs(), must be called before any other builder functions are called. This function completes the configuration of the IOC builder and initialisation of the builder's name space.

Parameters:
module_path This configures where ModuleVersion() will look for modules. By default the setting configured in paths is used.
record_names This configures the record naming convention. For examples see BasicRecordNames, the simplest format, TemplateRecordNames for naming templates, and DiamondRecordNames for normal IOC generation.
ioc_writer This configures how IOCs are written. Possible options are SimpleIocWriter for writing standalone databases or DiamondIocWriter for standard DLS IOCs.
dynamic_load This flag configures whether support modules are linked in or loaded at run-time. Probably now obsolete, as static linking is now the rule.
architecture Configures the target architecture. Must be set if an IOC is to be generated, not required for template only generation.
register_dbd Whether to call the register database function in the startup script. Like dynamic_load, probably obsolete.
simulation Controls whether simulation module definitions are to be selected.
epics_base Can be used to override the default selection of EPICS_BASE taken from the environment.
