Normal IOC builder initialisation function.

This function should normally be called before any other builder calls to complete configuration of the IOC builder. By default the vxWorks architecture is selected and the DLS record naming convention is used for record names; these can both be overridden.

Parameters:
architecture Architecture of target system, defaults to 'vxWorks-ppc604_long'
record_names Class used to define record naming convention, defaults to standard Diamond naming convention
**kargs See Configure() for other arguments that can be passed.

Function for configuring builder for template generation.

This function can be used to configure the IOC builder for template generation. By default record names are of the form $(DEVICE):<name> using the TemplateRecordNames() naming convention.

If this call is used to initialise the builder only database files can be written using the builder using WriteRecords().

Parameters:
record_names An alternative naming convention can be specified, the default is TemplateRecordNames().

def iocbuilder.configure.ParseAndConfigure (   options,
  dependency_tree = None 
)

Parse RELEASE files and setup iocbuilder using the information in them.

Parameters:
options The options object returned from ParseEtcArgs()
dependency_tree If present will be used to parse module dependencies from configure/RELEASE and other files.

Does the following:

  • If dependency_tree then:
    • Parse ../../configure/RELEASE
    • Substitute macros in <iocname>_RELEASE using ../../configure/RELEASE
    • Set options.epics_base
  • Run Configure(...) with the supplied information from options
  • If dependecy_tree then:
    • Do a libversion.ModuleVersion call for each module listed in the tree created from <iocname>_RELEASE and ../../configure/RELEASE

def iocbuilder.configure.ParseEtcArgs (   architecture = 'vxWorks-ppc604_long',
  epics_base = None 
)

An options parser for the standard gui options.

Parameters:
architecture Used to set the target architecture.
epics_base Can be used to override EPICS_BASE settings.

Does the following:

  • Parse sys.argv using the options parser
  • Return the options object, and args[1:]

The options object has the following attributes:

  • iocname: the name of the ioc that is to be built
  • iocpath: the path to the ioc directory
  • debug: if True, then debugging should happen
  • architecture: The architecture (e.g. linux-x86)
  • simarch: Equal to architecture if making a simulation, otherwise None
  • epics_base: Path to EPICS_BASE
